Protéger un tableau

Cette fiche pratique montre comment protéger un tableau de données en supprimant des cellules avec GaussSuppression. Pour ce faire on utilisera les données du package.

NoteRappels secret primaire et secondaire

Secret primaire : cellules supprimées car sensibles d’après les règles fixées (ex. fréquence, dominance). Secret secondaire : cellules supprimées pour empêcher la déduction du secret primaire à partir des marges du tableau.

On charge les packages nécessaires.

library(GaussSuppression)
library(dplyr)

1 Tableau de fréquences

Pour les tableaux de comptage on applique une règle de fréquence pour le secret primaire. Pour les statistiques d’entreprise à l’Insee, le seuil de fréquence est fixé à 3. Ainsi, toutes les cellules contenant 1 ou 2 individus doivent être supprimées.

Si on doit protéger un seul tableau de fréquence, alors il convient d’utiliser la fonction SuppressSmallCounts().

1.1 À partir des microdonnées

sprt_emp <- SSBtools::SSBtoolsData("sprt_emp")
head(sprt_emp)
     age      geo year ths_per
1 Y15-29    Spain 2014    66.9
2 Y15-29  Iceland 2014     1.8
3 Y15-29 Portugal 2014    11.6
4 Y30-64    Spain 2014   120.3
5 Y30-64  Iceland 2014     1.5
6 Y30-64 Portugal 2014    20.2
sprt_emp_protege <- SuppressSmallCounts(
  data = sprt_emp,
  maxN = 4, # seuil de fréquence, secret primaire
  dimVar = c("age","geo") # variables de croisement
)
[preAggregate 18*4->6*3]
[extend0 6*3->6*3]
GaussSuppression_anySum: ......
NoteArgument maxN

Si le seuil de fréquence est à 10, i.e. toutes les cellules qui regroupent moins de 10 individus sont en secret, alors il faut spécifier maxN = 9.

1.2 À partir des données pré-tabulées

Données pré-tabulées qui donnent le nombre d’accidents (freq) ventilé par la municipalité (mun) et la gravité (inj) de l’accident.

mun_accidents <- SSBtools::SSBtoolsData("mun_accidents")
head(mun_accidents)
  mun     inj freq
1  k1 serious    4
2  k2 serious    5
3  k3 serious    3
4  k4 serious    4
5  k5 serious    1
6  k6 serious    6

On regarde les modalités de chacune des variables de croisement et on remarque qu’il n’y a pas de totaux. C’est pour cela que l’on appelle ces données pré-tabulées, il manque les marges pour qu’on ait les données tabulées complètes.

unique(mun_accidents$mun)
[1] "k1" "k2" "k3" "k4" "k5" "k6"
unique(mun_accidents$inj)
[1] "serious" "light"   "none"    "unknown"

Les fonctions de suppression de GaussSuppression calculent les marges à partir des données pré-tabulées. On peut donc directement, protéger le tableau.

mun_acc_protege <- SuppressSmallCounts(
  data = mun_accidents,
  maxN = 2, # seuil de fréquence, secret primaire
  dimVar = c("mun","inj"), # variables de croisement
  freqVar = "freq" # fréquence
)
[extend0 24*3->24*3]
GaussSuppression_anySum: ...............

Suite à la protection des lignes ont été ajoutées dans les données.

nrow(mun_accidents)
[1] 24
nrow(mun_acc_protege)
[1] 35

En effet, il s’agit des totaux des deux variables de croisement qui ont été calculés par la fonction.

head(mun_acc_protege)

1 Total   Total   32   FALSE      FALSE
2 Total   light    3   FALSE       TRUE
3 Total    none    6   FALSE      FALSE
4 Total serious   23   FALSE      FALSE
5 Total unknown    0    TRUE       TRUE
6    k1   Total    4   FALSE       TRUE

De plus, on voit que deux colonnes ont été ajoutées.

ncol(mun_accidents)
[1] 3
ncol(mun_acc_protege)
[1] 5

La colonne primary est un booléen indiquant si la cellule est en secret primaire (de fréquence) ou non. La colonne suppressed est elle aussi un booléen qui indique quelles cellules doivent être supprimées (secret primaire et secret secondaire). Ainsi, toutes les cellules pour lesquelles primary = FALSE et suppressed = TRUE sont les cellules en secret secondaire.

Quand on regarde les premières lignes du masque, on voit que la cellule Total x unknown a une fréquence de 0 qui est en secret primaire. Par défaut SuppressSmallCounts() met en secret primaire les cellules qui valent zéro. Or, à l’Insee les fréquences qui valent 0 ne sont pas considérées comme sensibles et n’ont pas a être protégées par du secret primaire (ni même du secret secondaire).

Quand on considère que les zéros sont sensibles il y a 22 cellules en secret primaire.

mun_acc_protege %>% filter(primary) %>% nrow()
[1] 20

Pour que les fréquences à zéro ne soient pas en secret primaire on utilise l’argument protectZeros :

mun_acc_protege_pas_zero <- SuppressSmallCounts(
  data = mun_accidents,
  maxN = 2,
  dimVar = 1:2,
  freqVar = 3,
  protectZeros = FALSE # les zéros ne sont pas en secret primaire
)
[extend0 24*3->24*3]
GaussSuppression_anySum: .............................
mun_acc_protege_pas_zero %>% filter(primary)
  mun     inj freq primary suppressed
1  k2    none    1    TRUE       TRUE
2  k3   light    2    TRUE       TRUE
3  k3    none    1    TRUE       TRUE
4  k4   light    1    TRUE       TRUE
5  k5   Total    1    TRUE       TRUE
6  k5 serious    1    TRUE       TRUE

Il n’y a plus que 6 cellules en secret primaire.

Par défaut, les zéros sont des candidats au secret secondaire. Ainsi, même lorsque l’on exclut les zéros du secret primaire ils peuvent être cachés en secret secondaire. A l’Insee, on ne permet pas de supprimer les zéros même en secret secondaire car on fait l’hypothèse qu’il s’agit de zéros structurels. Ainsi, les utilisateurs des données peuvent facilement savoir quelles cellules ont une fréquence à zéro et pourraient donc déduire les informations des cellules que ces zéros protègent. Pour que les zéros ne soient pas des candidats au secret secondaire on utilise l’argument secondaryZeros = FALSE.

Warning

Quand les arguments protectZeros et secondaryZeros sont FALSE, alors les cellules qui ont une fréquence ou une valeur de zéro ne sont pas en secret primaire et secondaire. Or, à l’Insee on ne veut pas mettre de secret uniquement sur les cellules dont la fréquence vaut 0. Les cellules dont la fréquence différente de 0 et la valeur vaut 0 peuvent être protégées.

2 Tableau de volume

Pour diffuser un tableau qui présente un autre indicateur que simplement le nombre d’individus, par exemple le chiffre d’affaires des entreprises ou le revenu des ménages, alors il faut appliquer une règle supplémentaire : la règle de dominance.

D’après la règle de dominance 1 unité contributrice à une cellule ne peut contribuer à plus de k% de la valeur de celle-ci. Pour la statistique d’entreprise k = 85.

2.1 A partir des microdonnées

On utilise un extrait de microdonnées, base individuelle sur des établissements contenant les variables suivantes :

  • sector4 : secteur d’activité en 4 modalités
  • sector2 : secteur d’activité en 2 modalités
  • geo : pays de l’établissement
  • eu : établissement européenne ou non
  • company : groupe auquel appartient l’établissement
  • value : l’indicateur étudié (on peut imaginer qu’il s’agit du chiffre d’affaires)
magnitude1 <- SSBtools::SSBtoolsData("magnitude1")
head(magnitude1)
        sector4 sector2      geo    eu company value
1   Agriculture private Portugal    EU       A  75.9
2   Agriculture private Portugal    EU       B  24.5
3   Agriculture private    Spain    EU       A  96.6
4   Agriculture private    Spain    EU       B  43.2
5 Entertainment private  Iceland nonEU       B  16.8
6 Entertainment private Portugal    EU       B   7.1

Pour protéger un tableau de volumes on utilise la fonction SuppressDominantCells(). Dans un premier on protège un tableau qui ventile le chiffre d’affaires (value) par le secteur d’activité en 2 modalités (sector2) et le pays (geo).

magnitude1_protege <- SuppressDominantCells(
  data = magnitude1,
  n = 1,
  k = 85,
  numVar = "value",
  dimVar = c("sector2","geo")
)
[extraAggregate 20*3->5*5] Checking ....
GaussSuppression_numttHTT: ..........

Il y a moins de lignes dans le tableau protégé que dans les microdonnées puisque ces-dernières ont été agrégées.

nrow(magnitude1)
[1] 20
nrow(magnitude1_protege)
[1] 12

On voit qu’il y a un secret primaire de dominance sur le croisement public x Portugal.

1  public Portugal  23.6    TRUE       TRUE

Pour protéger ce secret primaire GaussSuppression supprimer trois autres cellules :

  • private x Portugal
  • public x Spain
  • private x Spain

3  public Portugal  23.6    TRUE       TRUE

2.1.1 Précisions sur le secret primaire de dominance

Pour l’instant on doit croire “aveuglement” GaussSuppression sur la façon dont il pause le secret primaire. On pourrait recalculer les dominances de chaque cellules pour connaître le pourcentage de dominance. Cependant, il n’est pas nécessaire de faire ce travail puisque l’argument allDominance nous permet de récupérer cette information.

magnitude1_protege_allDom <- SuppressDominantCells(
  data = magnitude1,
  n = 1,
  k = 85,
  numVar = "value",
  dimVar = c("sector2","geo"),
  allDominance = TRUE
)

2.1.2 Combiner règle de fréquence et règle de dominance

Dans les faits lorsque l’on protège un tableau de volume il faut appliquer les deux types de règles vues précédemment :

  • règle de fréquence sur le nombre d’individus par cellule
  • règle de dominance sur la valeur de la cellule
magnitude1_protege_freq_dom <- SuppressDominantCells(
  data = magnitude1,
  numVar = "value",
  dimVar = c("sector2","geo"),
  primary = c(DominanceRule, NContributorsRule), # utiliser règles freq et dom
  n = 1, # param dom
  k = 85, # param dom
  maxN = 2, # param freq
  allDominance = TRUE
)

Une fois que le tableau est protégé, on étudie la perte d’information. Un tableau de statistiques descriptives est osuvent suffisant, il nous donne une idée de la perte d’information résultant du processus de protection. Ici, on utilise stat_secret_GS() une fonction dont la code source se trouve dans le dossier fonction.

source("../fonctions/stats_secret_GS.R")
